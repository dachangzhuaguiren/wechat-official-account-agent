# 安全策略

## 报告漏洞

请不要在公开 Issue 中提交 API Key、访问码、文章内容或可直接利用的漏洞细节。

优先通过仓库的 [Private vulnerability reporting](https://github.com/k8w98rr595-blip/wechat-official-account-agent/security/advisories/new) 私密报告。报告应包含受影响版本、复现步骤、影响和建议修复方式，但不得包含真实 Secret。

## 支持范围

仅 `main` 分支最新部署版本接受安全修复。GitHub Pages 模式由用户在浏览器中临时提供自己的 DeepSeek Key；独立后端模式必须遵循 `docs/BACKEND_SECURITY_BOUNDARIES.md` 的部署检查。
